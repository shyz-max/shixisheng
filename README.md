# MES 生产领料申请编排工具

这是一个本地可用的流程配置工具，用来定义 MES 系统里“生产领料申请”的操作顺序。

## 打开方式

直接用浏览器打开 `index.html`。

如果必须使用 Internet Explorer 打开，页面已做 IE 兼容处理。Excel 上传解析依赖页面底部引用的 `xlsx` 库；内网不能访问外部网络时，可以直接从 Excel 复制 BOM 行并粘贴到 `粘贴 BOM 清单`。

## 能配置什么

- 系统登录：账号、密码、账号输入框、密码输入框、登录按钮、登录后等待时间。
- 流程步骤：第 1 步点哪里、第 2 步点哪里、输入什么、选择什么、等待多久。
- 定位方式：按钮文字、CSS、XPath、坐标、人工备注。
- BOM 领料：从 Excel 复制 BOM 清单，一次生成多个领料零件。
- Excel 导入：可直接上传 `.xlsx/.xls/.csv/.tsv`，按“零件图号/图号/物料编码”等列生成清单。
- 多零件申请：一个申请单可逐个选择多个零件、填写领料数量，再统一提交。
- 零件模板：零件编码、名称、数量、库位、工位、替代料、行号或自定义选择器。
- 最终动作：配置所有零件选完后点击哪个按钮，例如提交、保存、确认。
- 执行参数：浏览器类型、默认等待、失败处理、提交前确认。

## BOM 批量生成

在 `BOM 领料` 页签里，可以从 Excel 复制多行并粘贴到 `粘贴 BOM 清单`。

上传 Excel 时必须包含两列：

```text
图号    数量
```

列名可以在页面中配置，默认识别 `零件图号` 和 `数量`。也会自动兼容 `图号`、`物料编码`、`计划数`、`需求数量` 等常见列名。

粘贴 BOM 时支持列顺序：

```text
零件编码    零件名称    数量    库位    备注
```

点击 `按 BOM 生成零件` 会替换当前零件清单；点击 `追加到现有零件` 会把 BOM 零件加到已有清单后面。

上传或粘贴后，页面会自动生成 `待提交流程`，每个物料都会显示：图号、查询选择、提交添加、填写计划数。

如果在 BOM 选料前还需要点击页面按钮或填写单头字段，先到 `流程步骤` 页签配置 `申请单前置步骤`。这些步骤只执行一次，然后才开始按 Excel 清单逐个选择物料。也可以点击 `添加选料前模板` 生成一组可编辑的常用前置步骤。

执行器会按清单循环：

```text
点击添加/选择物料 -> 子表搜索零件图号 -> 精确选中同图号零件 -> 找不到则翻页继续找 -> 点击提交/确认添加 -> 回主表填写计划数 -> 下一个零件 -> 最后提交
```

如果 MES 还要求数量，可以额外配置 `数量输入框`，执行器会在每个零件确认前填写领料数量。

截图里的“选择物料”弹窗可以这样配：

- `每个零件先点击`：主表上的 `添加` 按钮
- `子表图号搜索框`：弹窗里的 `图号` 输入框
- `子表查询按钮`：弹窗里的 `查询` 按钮
- `搜索结果范围`：可留空，或填结果表格范围
- `子表下一页按钮`：弹窗底部分页的下一页按钮
- `子表确认添加按钮`：弹窗里的 `提交` 按钮
- `主表计划数输入框`：回到主表后当前行的计划数输入框

## 自动登录

在 `系统登录` 里填写：

- MES 系统账号和密码
- 账号输入框，例如 `#username`、`input[name='user']`
- 密码输入框，例如 `#password`、`input[type='password']`
- 登录按钮，例如 `登录`、`#loginBtn`、`//button[.='登录']`

执行器会先打开 MES 地址，自动输入账号密码并点击登录，然后继续执行后面的出库申请流程。

注意：导出的 JSON 会包含密码，请只在自己电脑上保存，不要上传带真实密码的配置文件。

## 保存与迁移

- `保存` 会把配置存在当前浏览器本机。
- `导出 JSON` 可以把配置文件发给别人或留档。
- `导入 JSON` 可以恢复以前保存的流程。

## 下一步可扩展

已经提供 Playwright 执行器模板。安装依赖后，把界面导出的 JSON 命名为 `mes-bom-issue-flow.json`，放在当前目录，然后运行：

```powershell
npm install
npm run run-flow
```

也可以指定其他配置文件：

```powershell
$env:MES_FLOW_CONFIG="C:\path\to\flow.json"
npm run run-flow
```
